**日期**: 2026-03-12
**版本**: v3.0
**状态**: ✅ 已确认

---

## 📱 核心设计理念

### 极简主义

```
扫码 → 设备操作页（作业/点检/维保）
```

**一个核心入口**: 扫码
**一个核心页面**: 设备操作页
**三个核心功能**: 作业、点检、维保

---

## 🎯 页面结构

```
/mobile
├── /page.tsx                    # 首页 = 扫码页（主入口）
├── /login/page.tsx              # 登录页
├── /equipment/
│   └── [id]/page.tsx            # 设备操作页（核心功能）
└── /inspection/
    ├── new/page.tsx             # 点检录入页
    └── [id]/page.tsx            # 点检详情页（可选）
```

---

## 🔄 用户流程

```
用户打开 H5
    │
    ▼
检查登录状态
    │
    ├── 未登录 ───> 登录页 ───> 登录成功 ───┐
    │                                       │
    └── 已登录 ──────────────────────────────┤
                                            ▼
                                       扫码页（首页）
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    │                       │                       │
                    ▼                       ▼                       ▼
                扫码识别                手动输入                 快捷入口
                    │                       │                       │
                    └───────────┬───────────┘                       │
                                │                                   │
                                ▼                                   ▼
                        设备操作页                          设备列表（可选）
                                │                                   │
                                ├──── 作业（开始/结束）                │
                                ├──── 点检（跳转录入）                │
                                └──── 维保（登记/完成）                │
```

## 🔧 技术架构

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Next.js 14+ | App Router |
| UI 库 | antd-mobile v5 | 移动端组件 |
| 扫码 | html5-qrcode | 摄像头扫码 |
| 状态 | React Context | AuthContext |
| 路由 | Next.js Router | 文件路由 |

### 布局组件

**MobileLayout**:
- 顶部 NavBar（带返回按钮）
- 无底部 TabBar
- 自动返回逻辑（设备操作页返回扫码页）

## 📊 功能对比

### 旧方案 vs 新方案

| 功能 | 旧方案 | 新方案 | 说明 |
|------|--------|--------|------|
| 首页 | 统计卡片 | 扫码页 | 简化入口 |
| TabBar | 3 个标签 | 无 | 减少干扰 |
| 扫码 | 独立页面 | 首页集成 | 一步到位 |
| 个人中心 | 独立 | 快捷入口 | 可选访问 |
| 设备列表 | 无 | 快捷入口 | 辅助功能 |

### 优势

1. **更简洁**: 用户打开直接扫码，没有多余信息
2. **更高效**: 减少页面层级，操作更直接
3. **更专注**: 聚焦核心功能（扫码→操作）
4. **更流畅**: 无 TabBar 切换，返回逻辑清晰

**方案 A**: 需要

**方案 B**: 不需要

- 实现：仅保留手动输入功能

**建议**: 方案 B（更简洁）

**建议**: 方案 B（简化移动端）
