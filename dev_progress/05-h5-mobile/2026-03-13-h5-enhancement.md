## 📋 修改内容

### 1. 点检转故障功能 ✅

**需求**: 点检发现异常时，询问用户是否上报故障

**实现**:
- 点检提交时自动统计异常项数量
- 如果有异常项，弹出对话框询问"是否上报故障"
- 选择"是" → 显示故障登记表单（故障等级、故障描述）
- 选择"否" → 直接提交点检记录

**流程**:
```
点检提交
    │
    ▼
检查异常项数量
    │
    ├── 有异常项 ───> 弹出询问对话框
    │                    │
    │                    ├── 选择"是" ───> 显示故障登记 ───> 提交点检 + 故障
    │                    │
    │                    └── 选择"否" ───> 仅提交点检
    │
    └── 无异常项 ───> 直接提交点检
```

**修改文件**:
- `eim/web/app/mobile/inspection/new/page.tsx`

**新增状态**:
```typescript
const [showFaultDialog, setShowFaultDialog] = useState(false);
const [showFaultRegister, setShowFaultRegister] = useState(false);
const [pendingInspectionData, setPendingInspectionData] = useState<any>(null);
```

**新增函数**:
- `submitInspectionOnly()` - 仅提交点检记录
- `submitInspectionWithFault()` - 提交点检记录并上报故障
- `showFaultRegisterDialog()` - 显示故障登记弹窗

**UI 组件**:
- 故障上报询问 Dialog（是/否选择）
- 故障登记 Dialog（故障等级、故障描述）

### 2. 维保完成结果选择 ✅

**需求**: 维保完成时，需要选择维保结果（全部解决/部分解决/未解决）

**实现**:
- **全部解决** → 设备状态变为 `standby`，无故障
- **部分解决** → 设备状态变为 `fault`，故障等级降低（需选择新等级）
- **未解决** → 设备状态保持 `fault`，故障等级不变

**修改文件**:
- `eim/web/app/mobile/equipment/[id]/page.tsx`

**维保结果选项**:
```typescript
[
  { label: '全部解决', value: 'resolved', color: '#52c41a' },
  { label: '部分解决', value: 'partially_resolved', color: '#faad14' },
  { label: '未解决', value: 'unresolved', color: '#ff4d4f' },
]
```

**表单逻辑**:
```typescript
// 全部解决
result = 'resolved'
→ 设备状态：standby
→ 故障等级：null

// 部分解决
result = 'partially_resolved'
→ 设备状态：fault
→ 故障等级：用户选择（降低等级）
→ 后续计划：必填

// 未解决
result = 'unresolved'
→ 设备状态：fault
→ 故障等级：保持不变
→ 后续计划：必填
```

**修改函数**:
- `handleCompleteMaintenance()` - 根据维保结果更新设备状态和故障等级

**UI 改进**:
- 维保结果选择器（Selector）
- 动态显示/隐藏故障等级和后续计划字段
- 使用 `shouldUpdate` 监听表单值变化

## 📝 注意事项

### 1. 点检转故障

- 故障上报是**可选操作**，不是所有异常都必须升级为故障
- 点检记录和故障记录**分别提交**，确保点检记录一定成功
- 故障来源标记为 `inspection`（点检中发现）

### 2. 维保完成

- 维保结果选择**影响设备状态和故障等级**
- "部分解决"和"未解决"时，**后续计划必填**
- 维保完成后，设备状态由后端 API 根据结果自动更新

### 3. 开始作业

- 设备类型判断使用**硬编码列表**（门式起重机、桥式起重机、装船机、卸船机）
- 如需新增设备类型，需修改代码
- 建议：未来可考虑配置化方案
