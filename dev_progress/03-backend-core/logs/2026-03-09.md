## 📋 今日完成

### 1. 核心业务规则更新 ✅

根据新的业务需求，更新了以下核心规则：

1. **作业流程完善**
   - 开始作业：填写船名、货品名称、操作人
   - 结束作业：记录作业时长、装卸吨位、可选故障登记

2. **点检异常转故障**
   - 点检发现异常时，可选择是否上报故障
   - 根据故障等级决定是否允许作业

3. **故障登记流程完善**
   - 照片上传功能（至少 1 张）
   - 故障描述必填
   - L1 故障实时企业微信通知

4. **维保完成确认流程**
   - 三种结果：全部解决/部分解决/未解决
   - 强制维保人签字确认
   - 可选验收人签字
   - 部分解决可降低故障等级

5. **作业与维保互斥**
   - 作业时不允许维保
   - 维保时不允许作业

## ⚠️ 遇到的问题及解决

### 问题 1: PostgreSQL 数组类型错误

**现象**: `ERROR: malformed array literal: ""`

**原因**: 数据库表结构使用 `text[]` 类型，但 Model 使用 `string` 类型

**解决**: 修改数据库表结构为 `text` 类型，存储 JSON 数组字符串

```sql
ALTER TABLE maintenance_record 
  ALTER COLUMN photos_before TYPE text,
  ALTER COLUMN photos_after TYPE text;
```

### 问题 2: SQL 语法错误

**现象**: `ERROR: syntax error at or near "?"`

**原因**: `GetUnresolvedFaults` 方法中 WHERE 子句重复使用 equipment_id

**解决**: 移除重复的 WHERE 条件

```go
// 修复前
.Where("equipment_id = ? AND status IN ?", []string{"open", "in_progress"})
if equipmentID > 0 {
    query = query.Where("equipment_id = ?", equipmentID)  // 重复
}

// 修复后
.Where("status IN ?", []string{"open", "in_progress"})
if equipmentID > 0 {
    query = query.Where("equipment_id = ?", equipmentID)
}
```
