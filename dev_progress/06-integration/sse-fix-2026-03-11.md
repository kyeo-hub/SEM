## 问题描述

Dashboard 页面加载时，SSE 连接失败，浏览器控制台显示：

```
GET http://localhost:8080/api/events/equipments?token=xxx 404 (Not Found)
SSE 连接错误：Event {isTrusted: true, type: 'error', ...}
```

## 问题原因

后端服务器启动时，SSE 路由 `/api/events/equipments` 没有被正确注册。

通过检查代码发现：
1. 路由配置正确：`internal/router/router.go` 中已配置 SSE 路由
2. Handler 初始化正确：`internal/router/init_handlers.go` 中已调用 `InitSSEHandler(equipmentSvc)`
3. 主函数调用顺序正确：`cmd/server/main.go` 中先 `InitializeHandlers` 再 `Setup`

**根本原因**：服务器进程在代码更新之前就已经启动，没有加载最新的路由配置。

## 解决方案

### 1. 重启后端服务器

```bash
# 停止现有服务器
pkill -f "cmd/server"

# 重新启动
cd /workspaces/SEM/eim
go run cmd/server/main.go
```

### 2. 验证路由注册

启动日志中应该看到：

```
[GIN-debug] GET    /api/events/equipments    --> github.com/kyeo-hub/eim/internal/handler.StreamEquipmentEvents (4 handlers)
```

### 3. 测试 SSE 端点

```bash
# 不带 token 测试（应返回 401）
curl -s http://localhost:8080/api/events/equipments
# 输出：{"code":401,"message":"未授权：未提供认证信息"}

# 带有效 token 测试（应返回 SSE 事件流）
curl -s -N "http://localhost:8080/api/events/equipments?token=YOUR_TOKEN"
# 输出：event:connected, event:equipments-init, event:equipments-update
```

## 预防措施

### 开发时的注意事项

1. **修改后端代码后必须重启服务器**
   - Go 是编译型语言，代码修改后需要重新编译和重启
   - 推荐使用 `air` 或 `gin` 等热重载工具

2. **检查路由注册日志**
   - 启动时查看 Gin 的路由注册日志
   - 确认所有路由都已正确注册

3. **使用健康检查端点**
   - `/health` 端点可以快速验证服务器是否正常运行

4. **SSE 连接调试技巧**
   - 使用 curl 测试 SSE 端点
   - 检查浏览器 Network 面板中的 EventSource 连接
   - 查看后端日志中的 SSE 连接/断开信息
