## 🧪 测试用例

### 测试 1: 开始作业 → 结束作业（无故障）

**测试步骤**:

1. 访问 H5 移动端：**http://localhost:3000/mobile**
2. 登录（如果需要）
3. 点击 **"扫码"** 或手动输入设备编号：`MQ40-02`
4. 点击 **"开始作业"** 按钮
5. 填写表单：
   - 船名：`测试船`
   - 货品名称：`铁矿石`
   - 操作人：`张三`
6. 点击 **"确认开始作业"**
7. 等待 Toast 提示"开始作业成功"
8. 设备状态变为 **作业中**（绿色）
9. 再次点击 **"结束作业"** 按钮
10. 填写表单：
    - 装卸吨位：`150.5`
    - 原因说明：`作业完成`
    - 是否有故障：`无故障`
    - 操作人：`张三`
11. 点击 **"确认"**

**预期结果**:
- ✅ Toast 提示"结束作业成功"
- ✅ 弹窗自动关闭
- ✅ 设备状态变为 **待命**（蓝色）
- ✅ 控制台日志显示：
  ```
  结束作业提交：{...}
  结束作业响应：{code: 0, message: "success", data: {...}}
  ```

**API 验证**:
```bash
# 获取今日作业记录
curl -H "Authorization: Bearer {token}" \
  "http://localhost:8080/api/operations/today?equipment_id=2"

# 应返回作业记录，包含 cargo_weight: 150.5
```

**预期结果**:

### 测试 3: 结束作业（有故障 - L1 禁止作业验证）

**前置条件**: 设备处于 L1 故障状态

**测试步骤**:

1. 扫码进入设备页面（设备状态应为故障，L1）
2. 点击 **"开始作业"** 按钮

**预期结果**:
- ❌ Toast 提示"设备严重故障，禁止作业！"
- ❌ 不允许开始作业

**API 验证**:
```bash
# 先设置设备为 L1 故障
curl -X PUT http://localhost:8080/api/equipments/2/status \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"status":"fault","fault_level_id":1}'

# 尝试开始作业（应该失败）
curl -X PUT http://localhost:8080/api/equipments/2/status \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"status":"working"}'

# 应返回：{"code":500,"message":"设备严重故障，禁止作业！"}
```
