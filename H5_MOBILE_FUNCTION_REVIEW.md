## 📱 当前架构

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Next.js 14+ | App Router |
| UI 库 | antd-mobile v5 | 移动端组件 |
| 扫码 | html5-qrcode | 摄像头扫码 |
| 状态 | React Context | AuthContext |
| 路由 | Next.js Router | 文件路由 |

### 页面结构

```
/mobile
├── /page.tsx              # 首页（设备统计 + 快捷入口）
├── /scan/page.tsx         # 扫码页（摄像头 + 手动输入）
├── /equipment/[id]/page.tsx  # 设备操作页（核心功能）
├── /inspection/
│   ├── new/page.tsx       # 点检录入页
│   └── [id]/page.tsx      # 点检详情页（待实现）
├── /profile/page.tsx      # 个人中心页
└── /login/page.tsx        # 登录页
```

### 布局组件

- **MobileLayout**: 统一布局（顶部导航 + 底部 TabBar）
- **TabBar**: 首页 / 扫码 / 我的 三个标签

## 🔧 技术问题

### 1. 图片上传

**现状**: 组件已实现，后端 API 未对接

**方案**:
- 使用 `input type="file"` 拍照/选择
- 使用 `canvas` 压缩图片
- 分片上传（大图片）
- 上传进度显示

### 2. 扫码优化

**现状**: 基础扫码可用

**方案**:
- 添加扫码成功提示音
- 支持相册选择二维码
- 扫码历史记录
- 手电筒开关

### 3. 签名优化

**现状**: Canvas 签名可用

**方案**:
- 添加签名清空功能
- 签名预览
- 签名压缩
- 支持手指和触控笔

### 4. 性能优化

**现状**: 页面加载较慢

**方案**:
- 图片懒加载
- 按需加载组件
- 虚拟列表（长列表）
- 缓存策略优化

## 🎨 UI/UX 建议

### 1. 色彩规范

| 状态 | 颜色 | 使用场景 |
|------|------|----------|
| 作业中 | #52c41a | 按钮、标签、统计 |
| 待命 | #1890ff | 按钮、标签、统计 |
| 维保 | #faad14 | 按钮、标签、统计 |
| 故障 | #ff4d4f | 按钮、标签、统计 |
| 主色 | #667eea | 渐变背景 |
| 辅色 | #764ba2 | 渐变背景 |

### 2. 字体规范

| 场景 | 大小 | 字重 |
|------|------|------|
| 标题 | 18-20px | Bold |
| 正文 | 14-16px | Normal |
| 辅助文字 | 12-13px | Normal |
| 统计数字 | 24-36px | Bold |

### 3. 交互规范

- **按钮点击**: 反馈时间 < 100ms
- **页面切换**: 动画时间 300ms
- **Toast 显示**: 2 秒自动消失
- **Dialog**: 点击遮罩关闭
- **加载**: 超过 1 秒显示 Loading

## 📝 开发建议

### 1. 代码组织

```
/mobile
├── /components/         # 移动端专用组件
│   ├── EquipmentCard    # 设备卡片
│   ├── StatusTag        # 状态标签
│   └── SignaturePad     # 签名组件
├── /hooks/              # 自定义 Hooks
│   ├── useCamera        # 摄像头 Hook
│   ├── useSignature     # 签名 Hook
│   └── usePullRefresh   # 下拉刷新 Hook
├── /utils/              # 工具函数
│   ├── imageCompress    # 图片压缩
│   └── storage          # 本地存储
└── /pages/              # 页面
```

### 2. 状态管理

**建议**: 使用 React Context + useReducer

```typescript
// MobileContext
{
  user: User | null,
  recentEquipments: Equipment[],
  scanHistory: string[],
  theme: 'light' | 'dark',
}
```

### 3. 网络请求

**建议**: 统一封装

```typescript
// mobileApi.ts
export const mobileApi = {
  getEquipment: (id: string) => ...,
  updateStatus: (id: string, status: string) => ...,
  submitInspection: (data: InspectionData) => ...,
  uploadImage: (file: File) => ...,
};
```

## ❓ 需要讨论的问题

### 1. 功能优先级

- 设备列表页是否必须？（扫码是主要入口）
- 点检记录列表优先级？（个人中心已有最近记录）
- 扫码历史是否需要？（浏览器历史可替代）

### 2. 技术选型

- 是否需要引入状态管理库？（Zustand/Jotai）
- 图片压缩使用哪个库？（browser-image-compression）
- 是否需要 PWA 支持？（离线使用）

### 3. UI/UX

- 底部 TabBar 是否需要 5 个标签？（增加"设备"和"记录"）
- 首页统计卡片是否需要更多数据？（维保、点检统计）
- 是否需要夜间模式？

### 4. 业务逻辑

- 点检记录是否允许修改/删除？
- 扫码历史保留多少条？
- 是否支持离线点检？（网络恢复后上传）
