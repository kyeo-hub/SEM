## 1. 项目概述

### 1.1 项目背景

本项目源于武汉钢铁集团物流有限公司的设备点检管理需求，旨在将传统的纸质设备点检本电子化、数字化，同时融合现有 SVM（特种车辆管理系统）的功能，打造一套完整的设备点检与运营管理系统。

### 1.2 核心需求

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R1 | 设备点检电子化（替代纸质点检本） | P0 |
| R2 | 保留 SVM 设备状态管理功能 | P0 |
| R3 | 统一二维码（一台设备一个码，包含所有状态入口） | P0 |
| R4 | H5 移动端点检录入（手机浏览器访问） | P0 |
| R5 | 设备故障分级管理 | P0 |
| R6 | 作业信息登记（船名、货品名称） | P1 |
| R7 | 企业微信机器人定时推送（故障设备日报） | P1 |
| R8 | 企业微信机器人维修完成通知 | P1 |
| R9 | 点检记录支持图片上传 | P1 |
| R10 | 地图展示（复用 SVM） | P1 |
| R11 | 大屏实时展示 | P1 |
| R12 | SVM 数据迁移 | P0 |

### 1.3 用户规模

- 预期用户：1000 人以内
- 并发预估：50-100 并发
- 设备数量：根据码头规模，预估 100-500 台

### 3.2 数据表定义

```sql
-- ==================== 用户表 ====================
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    password_hash   VARCHAR(100) NOT NULL,
    role            VARCHAR(20) DEFAULT 'inspector',  -- admin/inspector/supervisor/operator
    real_name       VARCHAR(50),
    department      VARCHAR(50),                      -- 部门/班组
    phone           VARCHAR(20),
    wechat_openid   VARCHAR(100),                     -- 微信小程序 openid
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ
);

-- ==================== 故障等级表 ====================
CREATE TABLE fault_level (
    id              SERIAL PRIMARY KEY,
    level_code      VARCHAR(10) UNIQUE NOT NULL,      -- L1/L2/L3
    level_name      VARCHAR(50) NOT NULL,             -- 严重故障/一般故障/轻微故障
    description     TEXT,                             -- 故障等级说明
    allow_work      BOOLEAN DEFAULT FALSE,            -- 是否允许带病作业
    color           VARCHAR(20) DEFAULT 'red',        -- 显示颜色
    sort_order      INT DEFAULT 0,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 初始化故障等级数据
INSERT INTO fault_level (level_code, level_name, description, allow_work, color, sort_order) VALUES
('L1', '严重故障', '设备完全不能作业，必须立即停机维修', FALSE, 'red', 1),
('L2', '一般故障', '设备可带病作业，但需尽快安排维修', TRUE, 'orange', 2),
('L3', '轻微故障', '设备可正常作业，建议安排维修', TRUE, 'yellow', 3);

-- ==================== 设备表 (扩展自 SVM vehicles) ====================
CREATE TABLE equipment (
    id              SERIAL PRIMARY KEY,
    code            VARCHAR(50) UNIQUE NOT NULL,      -- 设备编号
    name            VARCHAR(100) NOT NULL,            -- 设备名称 (如：40 吨门机 M4)
    type            VARCHAR(50) NOT NULL,             -- 设备类型 (门式起重机/桥式起重机/装船机/卸船机)
    company         VARCHAR(100),                     -- 使用单位 (如：外贸码头)
    location        VARCHAR(200),                     -- 安装位置
    latitude        DECIMAL(10, 8),                   -- 纬度 (地图展示)
    longitude       DECIMAL(11, 8),                   -- 经度 (地图展示)
    
    -- 状态相关
    status          VARCHAR(20) DEFAULT 'standby',    -- working/standby/maintenance/fault
    fault_level_id  INT REFERENCES fault_level(id),   -- 故障等级 (status=fault 时有效)
    
    -- 当前作业信息 (status=working 时填写)
    current_ship    VARCHAR(100),                     -- 当前船名
    current_cargo   VARCHAR(100),                     -- 当前货品名称
    
    -- QR 码 (统一二维码，包含设备所有状态入口)
    qr_code_uuid    VARCHAR(100) UNIQUE,              -- 二维码 UUID
    
    -- 点检配置
    inspection_enabled BOOLEAN DEFAULT TRUE,          -- 是否启用点检
    inspection_frequency VARCHAR(20) DEFAULT 'daily', -- 点检频次 (daily/weekly)
    
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ
);

CREATE INDEX idx_equipment_status ON equipment(status);
CREATE INDEX idx_equipment_type ON equipment(type);
CREATE INDEX idx_equipment_company ON equipment(company);

-- ==================== 点检标准表 ====================
CREATE TABLE inspection_standard (
    id              SERIAL PRIMARY KEY,
    equipment_type  VARCHAR(50) NOT NULL,             -- 设备类型 (关联 equipment.type)
    part_name       VARCHAR(50) NOT NULL,             -- 部位名称 (如：吊钩组、钢丝绳、卷筒组)
    part_order      INT DEFAULT 0,                    -- 部位排序
    item_name       VARCHAR(50) NOT NULL,             -- 点检项目 (如：吊钩、磨损)
    item_order      INT DEFAULT 0,                    -- 项目排序
    content         VARCHAR(100) NOT NULL,            -- 点检内容
    method          VARCHAR(20),                      -- 方法 (视/听/敲/测量/操作)
    limit_value     VARCHAR(200),                     -- 允许界限值
    is_required     BOOLEAN DEFAULT TRUE,             -- 是否必填项
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (equipment_type, part_name, item_name)
);

CREATE INDEX idx_inspection_standard_type ON inspection_standard(equipment_type);

-- ==================== 点检记录表 (主表) ====================
CREATE TABLE inspection_record (
    id              SERIAL PRIMARY KEY,
    equipment_id    INT NOT NULL REFERENCES equipment(id),
    inspection_date DATE NOT NULL,                    -- 点检日期
    shift           VARCHAR(10) NOT NULL,             -- 班次：before(班前)/during(班中)/handover(交班)
    inspector_id    INT REFERENCES users(id),         -- 点检人 ID
    inspector_name  VARCHAR(50),                      -- 点检人姓名 (冗余)
    
    -- 点检结果
    total_items     INT DEFAULT 0,                    -- 总点检项数
    normal_count    INT DEFAULT 0,                    -- 正常项数
    abnormal_count  INT DEFAULT 0,                    -- 异常项数
    overall_status  VARCHAR(10) DEFAULT 'normal',     -- normal/abnormal
    
    -- 问题记录
    problems_found  TEXT,                             -- 当班点检发现问题
    problems_handled TEXT,                            -- 班中问题处理
    legacy_issues   TEXT,                             -- 遗留问题及当班生产事务
    
    -- 签名

    -- 审核 (可选)
    reviewed_by     INT REFERENCES users(id),         -- 审核人
    reviewed_at     TIMESTAMPTZ,
    review_status   VARCHAR(20) DEFAULT 'pending',    -- pending/approved/rejected
    
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (equipment_id, inspection_date, shift)
);

CREATE INDEX idx_inspection_record_equip_date ON inspection_record(equipment_id, inspection_date);
CREATE INDEX idx_inspection_record_shift ON inspection_record(shift);

-- ==================== 作业信息历史表 ====================
CREATE TABLE operation_info_history (
    id              SERIAL PRIMARY KEY,
    equipment_id    INT NOT NULL REFERENCES equipment(id),
    ship_name       VARCHAR(100) NOT NULL,            -- 船名
    cargo_name      VARCHAR(100) NOT NULL,            -- 货品名称
    started_at      TIMESTAMPTZ NOT NULL,             -- 开始作业时间
    ended_at        TIMESTAMPTZ,                      -- 结束作业时间
    duration_minutes INT GENERATED ALWAYS AS (         -- 作业时长 (分钟)
        CASE WHEN ended_at IS NOT NULL 
        THEN EXTRACT(EPOCH FROM (ended_at - started_at))/60 
        ELSE NULL END
    ) STORED,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_operation_history_time ON operation_info_history(started_at);

    error_message   TEXT,
    sent_at         TIMESTAMPTZ,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_notification_type ON wechat_notification(notification_type);
CREATE INDEX idx_notification_status ON wechat_notification(status);

-- ==================== 视图：设备点检统计 ====================
CREATE VIEW v_equipment_inspection_stats AS
SELECT 
    e.id AS equipment_id,
    e.code AS equipment_code,
    e.name AS equipment_name,
    e.status AS current_status,
    DATE_TRUNC('month', CURRENT_DATE) AS stat_month,
    COUNT(DISTINCT ir.inspection_date) AS inspected_days,
    COUNT(DISTINCT CASE WHEN ir.overall_status = 'abnormal' THEN ir.inspection_date END) AS abnormal_days,
    COUNT(ir.id) AS total_records,
    SUM(ir.abnormal_count) AS total_abnormal_items
FROM equipment e
LEFT JOIN inspection_record ir ON e.id = ir.equipment_id 
    AND ir.inspection_date >= DATE_TRUNC('month', CURRENT_DATE)
WHERE e.deleted_at IS NULL
GROUP BY e.id, e.code, e.name, e.status;
```

## 6. H5 移动端设计

### 6.1 技术选型说明

**为什么选择 H5 而不是微信小程序？**

| 对比项 | 微信小程序 | H5 移动端 | 说明 |
|--------|-----------|----------|------|
| 开发周期 | 2 周 + 审核 1-2 周 | 1 周，无需审核 | H5 无需审核 |
| 人员管理 | 需微信授权登录 | 账号密码登录 | H5 更简单 |
| 扫码体验 | 原生扫码，体验好 | html5-qrcode，体验良好 | H5 可满足需求 |
| 图片上传 | 微信 API | input type=file | H5 更简单 |
| 跨平台 | 仅限微信内 | 任何浏览器 | H5 更灵活 |
| 更新 | 需审核 | 即时更新 | H5 即时生效 |

```typescript
// components/mobile/Scanner.tsx
import { Html5QrcodeScanner } from 'html5-qrcode';

export function Scanner({ onScan }: { onScan: (uuid: string) => void }) {
  useEffect(() => {
    const scanner = new Html5QrcodeScanner('reader', {
      fps: 10,
      qrbox: 250
    });
    
    scanner.render((decodedText) => {
      onScan(decodedText);
      scanner.clear();
    });
    
    return () => scanner.clear();
  }, []);
  
  return <div id="reader" />;
}
```

## 9. SVM 数据迁移方案

### 9.1 迁移映射表

| SVM 表 | EIM 表 | 迁移策略 |
|--------|--------|----------|
| `vehicles` | `equipment` | 字段映射 + 新增字段默认值 |
| `vehicle_status_history` | `equipment_status_history` | 直接迁移 + 新增 fault_level_id |
| `vehicle_status_segments` | (废弃) | 不迁移，用 history 表计算 |
| `daily_vehicle_stats` | (废弃) | 不迁移，用视图重新计算 |
| - | `fault_level` | 初始化插入 3 条数据 |
| - | `inspection_standard` | 按点检本初始化 |
| - | `inspection_record` | 无历史数据，从上线日开始 |

### 9.2 迁移脚本

```sql
-- ==================== 数据迁移脚本 ====================
-- 从 SVM 迁移到 EIM

-- 1. 迁移设备数据
INSERT INTO equipment (
    code, name, type, company, location,
    latitude, longitude, status,
    qr_code_uuid, inspection_enabled,
    created_at, updated_at
)
SELECT 
    code,
    name,
    '门式起重机' AS type,  -- 默认类型，可后续修正
    company,
    location,
    latitude,
    longitude,
    status,
    gen_random_uuid() AS qr_code_uuid,
    TRUE AS inspection_enabled,
    created_at,
    NOW() AS updated_at
FROM vehicles
WHERE deleted_at IS NULL;

-- 2. 迁移状态历史
INSERT INTO equipment_status_history (
    equipment_id, old_status, new_status,
    changed_by, reason, created_at
)
SELECT 
    e.id AS equipment_id,
    vsh.old_status,
    vsh.new_status,
    vsh.changed_by,
    vsh.reason,
    vsh.created_at
FROM vehicle_status_history vsh
JOIN equipment e ON vsh.vehicle_id = (
    SELECT id FROM vehicles WHERE code = e.code
);

-- 3. 创建迁移日志表
CREATE TABLE migration_log (
    id              SERIAL PRIMARY KEY,
    migration_name  VARCHAR(100) NOT NULL,
    status          VARCHAR(20) DEFAULT 'pending',
    records_migrated INT DEFAULT 0,
    error_message   TEXT,
    started_at      TIMESTAMPTZ DEFAULT NOW(),
    completed_at    TIMESTAMPTZ
);
```

### 9.3 迁移步骤

```bash
# 1. 备份原数据库
pg_dump -U svm_user svm_db > svm_backup_20260304.sql

# 2. 运行 EIM 初始化迁移
go run cmd/migrate/main.go --from-svm --svm-db="postgres://..." --eim-db="postgres://..."

# 3. 验证迁移结果
psql -U eim_user eim_db -c "SELECT COUNT(*) FROM equipment;"
psql -U eim_user eim_db -c "SELECT COUNT(*) FROM equipment_status_history;"

# 4. 生成迁移报告
go run cmd/migrate/main.go report
```

#### 操作 1: 🟢 开始作业

| 字段 | 说明 |
|------|------|
| 作业船舶 | 填写或选择船名 |
| 货品名称 | 下拉选择 (铁矿石/煤炭/钢材/矿粉/焦炭/...) |

→ 设备状态变更为 `working`，记录作业开始时间，大屏实时更新

#### 操作 2: 🔵 结束作业 (待命)

| 字段 | 说明 |
|------|------|
| 确认信息 | 显示当前作业信息、作业时长 |

→ 设备状态变更为 `standby`，记录作业结束时间，生成作业历史记录

#### 操作 3: 📋 点检录入

| 字段 | 说明 |
|------|------|
| 点检班次 | 选择：班前/班中/交班 |
| 点检表单 | 进入标准点检流程 (28 项) |

→ 提交点检记录，如有异常项自动触发故障报告

| 字段 | 说明 |
|------|------|
| 维保类型 | 日常维修/定期保养/故障维修/大修 |
| 维保内容 | 文字描述 |
| 故障等级 | L1/L2/L3 (仅故障维修时选择) |
| 上传图片 | 支持拍照上传 |

→ 设备状态变更为 `maintenance` 或 `fault`，推送企业微信通知

### D. 故障状态报告逻辑

故障不是在扫码后单独选择的，而是在以下操作中报告：

| 场景 | 操作路径 | 结果 |
|------|----------|------|
| **点检时发现异常** | 点检录入 → 某项选异常 → 填写故障等级 → 提交 | 设备状态→fault，推送企微 |
| **维保时报告故障** | 维保登记 → 故障维修 → 选故障等级 → 提交 | 设备状态→fault，推送企微 |
| **作业中发现故障** | 扫码 → 维保 → 故障维修 → 填写信息 | working→fault，推送企微 |
| **故障修复** | 扫码 → 维保 → 维修完成 → 填写内容 | fault→working/standby，推送企微 |
