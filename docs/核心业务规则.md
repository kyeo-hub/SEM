```
│ 点检标准   │ ✅       │ ❌         │ ❌           │
└────────────┴──────────┴────────────┴──────────────┘
```

### 规则 1: 设备状态管理

```
4 种状态：待命 (standby) | 作业 (working) | 维保 (maintenance) | 故障 (fault)

流转规则:
- 待命 ←→ 作业 ✅ (需通过故障等级检查)
- 作业 → 待命 ✅ (需填写作业记录)
- 维保 → 待命 ✅ (需维保完成确认)
- 故障 → 作业 ⚠️ (需通过故障等级检查)
- 维保 → 作业 ❌ (必须先设为待命)
- 作业 → 维保 ❌ (必须先结束作业)
```

### 规则 2: 故障等级作业限制

| 等级 | 名称 | 颜色 | 作业限制 | 通知方式 |
|------|------|------|----------|----------|
| L1 | 严重故障 | 🔴 红色 | ❌ **禁止作业** | 实时企业微信告警 |
| L2 | 一般故障 | 🟠 橙色 | ⚠️ **确认后允许** | 故障日报 |
| L3 | 轻微故障 | 🟡 黄色 | ⚠️ **确认后允许** | 故障日报 |

### 规则 3: 双重验证机制

```
前端验证 (第一道防线) → 用户体验，即时反馈
                          ↓
后端验证 (第二道防线) → 数据安全，防止绕过
```

### 规则 4: 一机一码原则

```
每台设备只有一个二维码
扫码后显示 4 个功能按钮：作业 | 待命 | 点检 | 维保
按钮状态根据设备当前状态动态变化
```

### 规则 5: 作业与维保互斥

```
⚠️ 安全红线:
- 作业时 ❌ 不允许维保
- 维保时 ❌ 不允许作业
- 必须先结束当前状态才能进入另一状态
```

## 0. 角色权限管理

### 0.1 角色定义

系统定义 3 种角色，每种角色有不同的权限范围：

### 0.3 角色权限后端验证

```go
// 中间件：角色权限验证
func RequireRole(roles ...string) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 从 Token 中获取用户角色
        userRole, exists := c.Get("user_role")
        if !exists {
            Error(c, http.StatusUnauthorized, "未登录")
            c.Abort()
            return
        }

        // 检查角色是否在允许列表中
        for _, role := range roles {
            if userRole == role {
                c.Next()
                return
            }
        }

        // 角色权限不足
        Error(c, http.StatusForbidden, "权限不足")
        c.Abort()
    }
}

// 使用示例
// 作业接口：只允许管理员和操作司机
router.PUT("/equipments/:id/start-work", RequireRole("admin", "operator"), StartWork)

// 维保接口：只允许管理员和维保员
router.PUT("/equipments/:id/start-maintenance", RequireRole("admin", "maintainer"), StartMaintenance)

// 用户管理：只允许管理员
router.GET("/users", RequireRole("admin"), GetUsers)
```

## 1. 设备状态管理

### 1.1 状态定义

| 状态码 | 状态名称 | 颜色 | 说明 |
|--------|----------|------|------|
| `standby` | 待命 | 🔵 蓝色 | 设备可用，等待作业 |
| `working` | 作业 | 🟢 绿色 | 设备正在作业中 |
| `maintenance` | 维保 | 🟡 黄色 | 设备正在维修/保养 |
| `fault` | 故障 | 🔴 红色 | 设备故障，不能作业 |

### 1.2 状态流转矩阵

| 当前状态 | 目标状态 | 是否允许 | 条件/说明 |
|----------|----------|----------|-----------|
| standby | working | ✅ | 需通过故障等级检查 |
| standby | maintenance | ✅ | 直接变更 |
| standby | fault | ✅ | 需指定故障等级 |
| working | standby | ✅ | **需填写作业记录**（时长/吨位/可选故障） |
| working | maintenance | ❌ | **必须先结束作业** |
| working | fault | ✅ | 作业中发现故障（可选） |
| maintenance | standby | ✅ | **需维保完成确认**（签字） |
| maintenance | working | ❌ | **必须先设为待命** |
| maintenance | fault | ✅ | 维保中发现故障 |
| fault | standby | ✅ | 故障全部解决 |
| fault | working | ⚠️ | 需通过故障等级检查 |
| fault | maintenance | ✅ | 故障转维修（部分解决） |

### 1.3 状态流转图

```
                    ┌──────────────┐
                    │   standby    │
                    │    待命      │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │ 开始作业    │ 结束作业    │
              ▼            │            │

       │   作业     │                   │
```

### 2.2 作业限制判断逻辑

```
开始作业请求
    │
    ▼
┌─────────────────────────┐
│ 1. 检查当前状态          │
│    status = maintenance? │
│    → 是：❌ 禁止作业     │

│      - L1 (false) → ❌   │
│      - L2/L3 (true) → ⚠️ │
│    → 否：继续检查       │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│ 3. 允许作业              │
│    → ✅ 开始作业         │
└─────────────────────────┘
```

### 2.3 前端验证代码

```typescript
// 文件：web/app/mobile/equipment/[id]/page.tsx
const checkAllowWork = () => {
  // 1. 维保状态 → 禁止作业
  if (equipment.status === 'maintenance') {
    Toast.show('设备正在维保中，禁止作业');
    return;
  }

  // 2. 故障状态 → 检查故障等级
  if (equipment.status === 'fault' && equipment.fault_level) {
    if (!equipment.fault_level.allow_work) {
      // L1 故障 → 禁止作业
      Toast.show('设备严重故障，禁止作业！');
      return;
    } else {
      // L2/L3 故障 → 允许但需确认
      Dialog.confirm('设备故障，允许带病作业，是否继续？');
      return;
    }
  }

  // 3. 待命状态 → 允许作业
  setWorkModalVisible(true);
};
```

```go
// 结束作业处理
func (s *EquipmentService) EndWork(ctx context.Context, id int64, req *EndWorkRequest) error {
    // 1. 创建作业记录
    operation := &model.OperationRecord{
        EquipmentID:    id,
        ShipName:       req.ShipName,
        CargoName:      req.CargoName,
        CargoWeight:    req.CargoWeight,
        DurationMinutes: req.Duration,
        StartTime:      req.StartTime,
        EndTime:        time.Now(),
        OperatorName:   req.Operator,
        HasFault:       req.HasFault,
    }
    
    // 2. 如果有故障，创建故障记录并更新设备状态
    if req.HasFault && req.FaultLevelID > 0 {
        fault := &model.FaultRecord{
            EquipmentID:    id,
            FaultLevelID:   req.FaultLevelID,
            Description:    req.FaultDescription,
            ReporterName:   req.Operator,
            Source:         "operation",  // 来源于作业
        }
        s.faultRepo.Create(ctx, fault)
        
        // 更新设备状态为故障
        s.equipmentRepo.UpdateStatus(ctx, id, "fault", &req.FaultLevelID)
        
        // 发送企业微信通知
        if req.FaultLevelID == 1 {
            s.wechatBot.SendFaultAlert(ctx, fault)
        }
    } else {
        // 无故障，更新设备状态为待命
        s.equipmentRepo.UpdateStatus(ctx, id, "standby", nil)
    }
    
    // 3. 保存作业记录
    s.operationRepo.Create(ctx, operation)
    
    return nil
}
```

### 6.2 维保完成确认

**必填信息**：
| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 维保结果 | 全部解决/部分解决/未解决 | ✅ 必填 |
| 维保人签字 | 电子签名 | ✅ 必填 |
| 维保内容 | 实际完成的维保工作 | ✅ 必填 |
| 故障等级 | 维保后的故障等级 | ⚠️ 部分解决时必填 |
| 后续计划 | 未解决部分的后续安排 | ⚠️ 未解决时必填 |
| 验收人 | 维保验收人员 | ⚠️ 可选 |
| 验收人签字 | 验收人电子签名 | ⚠️ 可选 |

**维保结果说明**：
| 结果 | 设备状态变更 | 说明 |
|------|--------------|------|
| 全部解决 (resolved) | → standby | 故障完全修复，设备恢复待命 |
| 部分解决 (partially_resolved) | → fault (降级) | 部分修复，可降低故障等级 |
| 未解决 (unresolved) | → fault (保持) | 故障未修复，保持原状态 |

**前端实现**：
```typescript
// 维保完成确认弹窗
interface MaintenanceComplete {
  result: 'resolved' | 'partially_resolved' | 'unresolved';
  fault_level_id?: number;  // 部分解决时必填
  maintainer_name: string;  // 必填
  maintainer_signature: string;  // 必填，电子签名
  content: string;  // 必填，实际完成的维保工作
  next_plan?: string;  // 未解决时必填
  acceptor_name?: string;  // 可选
  acceptor_signature?: string;  // 可选
}

// 提交处理
const handleMaintenanceComplete = async (values: MaintenanceComplete) => {
  // 1. 根据维保结果更新设备状态
  let newStatus: string;
  let faultLevelId: number | null = null;
  
  switch (values.result) {
    case 'resolved':
      newStatus = 'standby';
      break;
    case 'partially_resolved':
      newStatus = 'fault';
      faultLevelId = values.fault_level_id;  // 降低故障等级
      break;
    case 'unresolved':
      newStatus = 'fault';
      faultLevelId = equipment.fault_level_id;  // 保持原故障等级
      break;
  }
  
  // 2. 更新设备状态
  await fetch(`/api/equipments/${equipmentId}/status`, {
    method: 'PUT',
    body: JSON.stringify({
      status: newStatus,
      fault_level_id: faultLevelId,
      reason: '维保完成确认'
    })
  });
  
  // 3. 保存维保记录
  await fetch('/api/maintenance/complete', {
    method: 'POST',
    body: JSON.stringify(values)
  });
};
```

## 7. 作业与维保互斥规则

### 7.1 互斥规则说明

```
⚠️ 安全红线：作业与维保互斥

规则 1: 作业时不允许维保
- 设备状态 = working 时，维保登记按钮禁用
- 提示："设备正在作业中，请先结束作业"

规则 2: 维保时不允许作业
- 设备状态 = maintenance 时，开始作业按钮禁用
- 提示："设备正在维保中，禁止作业"
```

### 7.2 前端实现

```typescript
// 设备操作页按钮状态控制

    case 'working':

        work: { disabled: false, text: '结束作业' },
        standby: { disabled: true, text: '作业中' },
        maintenance: { disabled: true, text: '作业中' },
        fault: { disabled: true, text: '作业中' }
      };
    
    case 'maintenance':
      return {
        work: { disabled: true, text: '维保中' },
        standby: { disabled: true, text: '维保中' },
        maintenance: { disabled: false, text: '维保完成' },
        fault: { disabled: true, text: '维保中' }
      };
    
    case 'fault':
      return {
        work: { disabled: !equipment.fault_level?.allow_work, text: '开始作业' },
        standby: { disabled: false, text: '设为待命' },
        maintenance: { disabled: false, text: '维保登记' },
        fault: { disabled: true, text: '故障中' }
      };
    
    default: // standby
      return {
        work: { disabled: false, text: '开始作业' },
        standby: { disabled: true, text: '待命' },
        maintenance: { disabled: false, text: '维保登记' },
        fault: { disabled: false, text: '故障登记' }
      };
  }
};
```
