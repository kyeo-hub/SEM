## 核心设计要点

### 1. 统一二维码设计

**变更前**: 每台设备每个状态一个二维码

**变更后**: 每台设备一个统一二维码，扫码后选择 4 种操作

```
┌─────────────────────────────────────────────────────────────────┐
│                     设备操作页                                   │
│                                                                 │
│   🟢 40 吨门机 M4                                                │
│      外贸码头 · 门式起重机                                       │
│   当前状态：作业中  船名：长航 5 号  货品：铁矿石                  │
│                                                                 │
│   ┌───────────┐   ┌───────────┐   ┌───────────┐   ┌───────────┐│
│   │  🟢 作业   │   │  🔵 待命   │   │  📋 点检   │   │  🔧 维保   ││
│   │  开始作业  │   │  结束作业  │   │  点检录入  │   │  维保登记  ││
│   └───────────┘   └───────────┘   └───────────┘   └───────────┘│
└─────────────────────────────────────────────────────────────────┘
```

### 2. 故障报告逻辑

故障不是独立选项，而是在以下场景中报告：

| 场景 | 操作路径 | 故障等级选择 |
|------|----------|--------------|
| 点检异常 | 点检录入 → 某项选异常 → 填写描述 | L1/L2/L3 |
| 维保故障 | 维保登记 → 故障维修 → 填写内容 | L1/L2/L3 |
| 作业中发现 | 扫码 → 维保 → 故障维修 | L1/L2/L3 |
| 故障修复 | 扫码 → 维保 → 维修完成 | 自动恢复 |

### 3. 故障分级

| 等级 | 名称 | 颜色 | 作业判定 | 推送策略 |
|------|------|------|----------|----------|
| L1 | 严重故障 | 红色 | 禁止作业 | 立即推送 + 电话通知 |
| L2 | 一般故障 | 橙色 | 可带病作业 | 立即推送 |
| L3 | 轻微故障 | 黄色 | 正常作业 | 日报汇总 |

### 4. 作业信息跟踪

```sql
-- 设备表字段
current_ship    VARCHAR(100)   -- 当前船名
current_cargo   VARCHAR(100)   -- 当前货品名称
status          VARCHAR(20)    -- working/standby/maintenance/fault

-- 作业历史表
operation_info_history
├── ship_name       -- 船名
├── cargo_name      -- 货品名称
├── started_at      -- 开始时间
├── ended_at        -- 结束时间
└── duration_minutes -- 作业时长
```

### 5. 企业微信推送

**场景 1: 故障设备日报**
- 时间：每日 08:00
- 内容：当前故障设备列表、故障等级、处理建议
- 接收人：维修班组、管理人员

**场景 2: 故障修复通知**
- 触发：设备状态从 fault→working/standby
- 内容：设备名称、故障等级、维修内容、停机时长
- 接收人：维修班组、管理人员

## 核心业务规则摘要

### 设备状态管理

**4 种状态**：`standby`（待命）、`working`（作业）、`maintenance`（维保）、`fault`（故障）

**状态流转约束**：
- 维保状态 → 禁止作业
- 故障状态 → 需通过故障等级检查才能作业
- 作业中 → 不能直接转维保，必须先结束作业

### 故障等级与作业限制

| 等级 | 名称 | allow_work | 作业限制 |
|------|------|------------|----------|
| L1 | 严重故障 | false | ❌ 禁止作业 |
| L2 | 一般故障 | true | ⚠️ 确认后允许 |
| L3 | 轻微故障 | true | ⚠️ 确认后允许 |

### 双重验证机制

**前端验证**（用户体验）→ 即时反馈，避免无效请求
**后端验证**（数据安全）→ 防止恶意绕过，最终安全防线

### 一机一码原则

- 每台设备只有一个二维码
- 扫码后显示 4 个功能按钮（作业/待命/点检/维保）
- 按钮状态根据设备当前状态动态变化
